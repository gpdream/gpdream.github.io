maven详解

  • Making the build process easy 使编译更简单，一句maven命令就能编译掉整个项目，可以把包含的类似代码混淆，前端代码压缩等都包含进去。
  • Providing a uniform build system 提供统一编译环境，在pom.xml里指定编译需要的环境，例如java/nodejs等的版本，项目的自动依赖等

    maven是什么

    maven能够帮忙解决的问题:

  • Making the build process easy 使编译更简单，一句maven命令就能编译掉整个项目，可以把包含的类似代码混淆，前端代码压缩等都包含进去。
  • Providing a uniform build system 提供统一编译环境，在pom.xml里指定编译需要的环境，例如java/nodejs等的版本，项目的自动依赖等
  • Providing quality project information提供高质量的项目信息，依赖列表，开发者信息，邮箱等都可以通过maven的pom中维护
  • Providing guidelines for best practices development提供最佳实践开发的指南。（单元测试），问题单跟踪(jira集成)，持续继承(CI集成)等
  • Allowing transparent migration to new features 透明更新新功能，例如有插件需要node js的0.4.0版本，现在要升级成5.0.0可以在pom中直接修改。

maven的误解

  • maven是一个站点和文档工具（只是拿来做文档管理）
  • maven是一个ant扩展，拿来做依赖包的下载
  • maven是一套可复用的ant脚本

maven主要功能汇总

  • 项目或module的快速构建，一行mvn archetype:generate命令就能构建出一个项目的骨架。
  • 所有项目环境一致，新加入的开发人员无需投入时间
  • 上下级依赖管理，一个项目中可能有多个module，可能会有parent。
  • 能同时作用在多个项目上
  • 庞大的一来仓库(reoisutory)，实时跟进开源项目的更新
  • 可轻易扩展的java或脚本语言的插件
  • 立即maven增加新功能，没有或者很少会有额外配置
  • ant以外的依赖管理和部署管理(ant是一个java build的工具，maven除了兼容ant的功能以外还能做很多其他事情)
  • 编译打包，一行maven命令就可以打包成jar,war,zip等包，无需其他命令
  • 站点项目信息管理，maven能生成一个web site或者pdf把你想要的项目信息添加进去。
  • 版本管理和发布管理：maven能够把jira,ci,git等继承在一起，将编译好的包直接发布到CI之类的地方
  • 依赖管理，jar包之类的依赖

maven的生命周期

maven中的每一个步骤都是生命周期来构建的。比如如下生命周期(事实上生命周期远多于此，光是clean一个动作就有3个阶段)：

  • validate ：校验项目是否正确可使用
  • compile: 编译当前项目的源码
  • test: 执行单元测试，看是否正常
  • package :把编译好的项目打包
  • verify 运行集成测试，看打包是否符合预期
  • install 打好的包放到本地仓库中，可以给本地其他项目提供依赖
  • deploy 把编译好的包上传到远程仓库

当我们运行命令mvn install的时候，这个命令默认会包含生命周期 validate,compile,test,package,verify,install。如果有一步失败了，就会失败。

maven的插件

maven有很多插件，此处说几个核心插件。maven的插件非常非常多，有一百万个那么多。

  • clean:build结束之后清空
  • compiler:编译java源码
  • deploy:把一个artifact部署到远程仓库上
  • failsafe: 集成测试
  • install:把一个artifcat部署到本地仓库上
  • resources:资源拷贝到指定目录，比如jar包输出目录
  • site: 生成项目的站点文档
  • surefire: 单元测试

pom

pom全称Project Object Model，maven项目所有的配置项都可以填在pom.xml中。文章最后会贴一个pom.xml所有配置项的意思。

profile

profile 是一个可选的可触发的配置。profile可以在pom.xml中声明或者在maven的settings.xml配置文件里声明。

触发profile

profile可以通过几种方式来触发.

  • 明确声明，类似命令 mvn -p profile-1这种方式来触发profile-1.

  • 在maven settings里说明触发

  • 根据系统环境变量触发，例如当jdk版本是1.3或者1.6的时候

  • 根据命令参数触发

如上配置，那当运行mvn -Ddebug=false的时候，就会触发对应profile

  • 根据缺少文件触发

像我们的npm install和bower install就可以这么做，如果missing了bower compentes文件夹就自动触发bower install。
